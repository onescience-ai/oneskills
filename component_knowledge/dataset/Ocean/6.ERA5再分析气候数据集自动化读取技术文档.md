## 一、数据集核心基础信息（代码生成前置知识）

### 1.1 数据集唯一标识

全称：ERA5 全球大气再分析数据集

唯一主键：dataset_id + variable + time_step（组合键）

数据类型：全球气候再分析网格数据（大气、陆面、海洋相关变量）

官方数据源：ECMWF / Copernicus Climate Data Store（CDS）

数据规模：时间范围自 1940 年至今，逐小时产品，全球约 31 km 网格、137 个垂直层。

补充必要项：需记录数据版本、下载日期、变量清单、时间范围与坐标系。

### 1.2 数据对象与解析优先级

| 数据对象 | 代码标识 | 解析优先级 | 处理逻辑 |
| --- | --- | --- | --- |
| 单层变量 | SINGLE_LEVEL | 高 | 默认全量解析，优先处理 2m 温度、风、降水 |
| 压强层变量 | PRESSURE_LEVEL | 高 | 按层级展开 |
| 模式层变量 | MODEL_LEVEL | 中 | 分层读取，避免一次性载入全部层 |
| 集合不确定性变量 | ENSEMBLE | 中 | 按半分辨率单独保存 |

## 二、数据格式规范与字段映射（必解析字段清单）

### 2.1 主格式

主格式：GRIB（.grib）—— 官方归档优先格式。按消息块解析，禁止按行读取。

| 字段名 | 代码变量名 | 数据类型 | 是否必填 | 缺失处理 |
| --- | --- | --- | --- | --- |
| shortName | var_name | 字符串 | 是 | 终止解析 |
| dataDate | date | 整数 | 是 | 跳过当前消息 |
| dataTime | time | 整数 | 是 | 跳过当前消息 |
| level | level | 整数 | 否 | 单层变量赋值 0 |
| values | values | 数组 | 是 | 剔除损坏消息 |

### 2.2 配套格式

配套格式：NetCDF（.nc）。CDS 分发常用格式，按变量维度 time / level / lat / lon 解析。

| 行标识/字段 | 字段含义 | 代码变量名 |
| --- | --- | --- |
| 维度变量 | 时间与空间坐标 | coords |
| 数据变量 | 气象场值 | data_var |

### 2.3 兼容/辅助格式

兼容/辅助格式：ZIP / CDS API 请求 JSON。用于批量下载与任务配置，不做数值解析。

## 五、解析逻辑与遍历规则（分层执行步骤）

### 5.1 单条目解析步骤（智能体按顺序生成代码）

下载 GRIB/NetCDF → 初始化读取器 → 校验变量与坐标维度 → 读取单时间片 → 生成统计摘要 → 输出标准 JSON。

### 5.2 多对象处理规则

同一变量按时间块顺序读取；禁止一次性载入全年高维数据到内存。

## 六、校验规则与异常处理（判断条件+分支逻辑）

### 6.1 必做校验（代码必须实现，顺序执行）

时间坐标必须单调递增

纬度范围应在 [-90,90]

经度应在 [0,360] 或 [-180,180] 内统一一种

缺测值必须映射为 NaN

### 6.2 异常处理分支（固定捕获+处理逻辑）

| 异常类型 | 触发场景 | 代码处理逻辑 |
| --- | --- | --- |
| ValueError | 维度不匹配 | 关闭当前文件并记录 |
| OSError | GRIB/NetCDF 文件损坏 | 跳过当前文件 |
| RequestException | CDS 下载失败 | 重试 3 次 |
| MemoryError | 多维数组过大 | 按 time chunk 分块读取 |

## 七、数据过滤与清洗规则（剔除条件清单）

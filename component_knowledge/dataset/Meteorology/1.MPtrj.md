## 一、数据集核心基础信息（代码生成前置知识）

### 1.1 数据集唯一标识

全称：Materials Project Trajectory（MPtrj）数据集

唯一主键：mp_id + frame_id（组合键）

数据类型：材料结构弛豫轨迹、单点能量、力与应力标签

官方数据源：Materials Project / Figshare / Materials Project Docs

数据规模：清洗版约 1.58M structures；来源于 MP 中 PBE GGA 与 GGA+U 任务轨迹。

补充必要项：需记录数据发布日期、任务来源、能量单位与结构版本。

### 1.2 数据对象与解析优先级

| 数据对象 | 代码标识 | 解析优先级 | 处理逻辑 |
| --- | --- | --- | --- |
| 弛豫轨迹帧 | RELAX_FRAME | 高 | 默认全量解析，保留步骤顺序 |
| 单点静态帧 | STATIC_FRAME | 中 | 只保留最终标签 |
| GGA 任务 | GGA | 高 | 默认优先解析 |
| GGA+U 任务 | GGA_U | 中 | 保留泛函标签 |

## 二、数据格式规范与字段映射（必解析字段清单）

### 2.1 主格式

主格式：JSON（结构字典）—— 代码优先解析。每个 mp-id 下包含多个 frame 字典。

| 字段名 | 代码变量名 | 数据类型 | 是否必填 | 缺失处理 |
| --- | --- | --- | --- | --- |
| structure | structure | 字典 | 是 | 终止解析 |
| energy_per_atom | energy | 浮点数 | 是 | 跳过当前帧 |
| force | forces | 数组 | 是 | 剔除当前帧 |
| stress | stress | 数组 | 否 | 赋值为空 |
| magmom | magmom | 数组/空 | 否 | 保留为空 |

### 2.2 配套格式

配套格式：Pymatgen Structure 字典。按 lattice / sites / species 解析。

| 行标识/字段 | 字段含义 | 代码变量名 |
| --- | --- | --- |
| lattice | 晶格矩阵 | lattice |
| sites | 原子位点 | sites |

### 2.3 兼容/辅助格式

兼容/辅助格式：ZIP / JSON 压缩包。大文件建议流式解压后逐对象读取。

## 六、校验规则与异常处理（判断条件+分支逻辑）

### 6.1 必做校验（代码必须实现，顺序执行）

mp_id 非空

structure 字典可被 pymatgen 正常解析

forces 维度应与原子数一致

energy_per_atom 必须为有限值

### 6.2 异常处理分支（固定捕获+处理逻辑）

| 异常类型 | 触发场景 | 代码处理逻辑 |
| --- | --- | --- |
| JSONDecodeError | JSON 文件损坏 | 跳过当前对象 |
| ValueError | Structure 字典非法 | 关闭当前帧并记录 |
| FileNotFoundError | 压缩包缺失 | 跳过当前批次 |
| MemoryError | 大 JSON 内存溢出 | 流式解析，仅保留当前材料 |

## 八、批量调度与存储规范（路径+命名+格式）

### 8.1 目录规范（代码自动创建，禁止修改路径）

原始文件缓存：./mptrj/json/{split}.json 或 ./mptrj/archives/{split}.zip；解析结果：./storage/{首字母}/{mp_id}_result.json；日志：./logs/{YYYYMMDD}.log。

### 8.2 结果格式规范（固定JSON结构）

```json
{
"mp_id": "mp-1234",
"n_frames": 12,
"frames": [
{
"frame_id": "mp-1234-0",
"energy_per_atom": -3.51,
"n_atoms": 40
}
]
}
```

### 8.3 批量调度规则

FIFO 顺序执行

已存在有效结果时跳过重复解析

执行状态：待处理=0、解析中=1、已完成=2、失败=3

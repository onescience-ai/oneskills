## 一、数据集核心基础信息（代码生成前置知识）

### 1.1 数据集唯一标识

全称：Evo 2 / OpenGenome2 预训练数据集

唯一主键：sample_id（字符串，代码内统一转为大写）

数据类型：DNA/RNA/蛋白质序列预训练样本与配套元信息

官方数据源：Arc Institute / GitHub 发布的 Evo 2 模型与 OpenGenome2 数据说明

数据规模：OpenGenome2 约 8.8 万亿 tokens，模型支持最长约 1,000,000 bp 上下文。

补充必要项：需记录模型版本、checkpoint 名称、tokenizer 版本和发布时间。

### 1.2 数据对象与解析优先级

| 数据对象 | 代码标识 | 解析优先级 | 处理逻辑 |
| --- | --- | --- | --- |
| 原核基因组片段 | PROKARYOTE | 高 | 默认全量读取，保留物种与片段坐标 |
| 真核基因组片段 | EUKARYOTE | 高 | 按长上下文切片读取 |
| 病毒/噬菌体片段 | VIRAL | 中 | 保留来源标签与宿主信息 |
| 下游设计样本 | DESIGN | 中 | 记录生成输入与输出序列 |

## 二、数据格式规范与字段映射（必解析字段清单）

### 2.1 主格式

主格式：纯文本序列切片（.txt / .jsonl）—— 代码优先解析。格式类型：按样本行或 JSON Lines 存储序列及元信息。

| 字段名 | 代码变量名 | 数据类型 | 是否必填 | 缺失处理 |
| --- | --- | --- | --- | --- |
| sample_id | sample_id | 字符串 | 是 | 终止解析 |
| sequence | sequence | 字符串 | 是 | 跳过样本 |

| 行标识/字段 | 字段含义 | 代码变量名 |
| --- | --- | --- |
| > | 序列ID | seq_id |
| 序列行 | 碱基或氨基酸序列 | sequence |

### 2.3 兼容/辅助格式

兼容/辅助格式：JSON / JSONL（元信息、打分结果、采样配置）。记录物种标签、长度、分片边界与采样参数。

## 六、校验规则与异常处理（判断条件+分支逻辑）

### 6.1 必做校验（代码必须实现，顺序执行）

sample_id 非空且唯一

序列字符仅允许 A/C/G/T/N 或合法氨基酸字符

序列长度 ≤ 1,000,000

JSONL 每行必须能独立反序列化

### 6.2 异常处理分支（固定捕获+处理逻辑）

| 异常类型 | 触发场景 | 代码处理逻辑 |
| --- | --- | --- |
| JSONDecodeError | 元信息行损坏 | 跳过当前行并记录偏移量 |
| UnicodeDecodeError | 文本编码异常 | 按 UTF-8 失败后回退 latin-1，只读一次 |
| FileNotFoundError | 样本文件缺失 | 记录失败并跳过 |
| MemoryError | 长序列或大批量内存溢出 | 分片读取，仅保留当前批次 |

## 七、数据过滤与清洗规则（剔除条件清单）

过滤空序列：长度为 0 的样本直接跳过

过滤非法字符样本：出现未定义 token 时剔除

过滤重复 sample_id：默认保留首条

过滤过短片段：长度 < 32 的样本不进入训练集

FIFO 顺序执行

执行状态：待处理=0、解析中=1、已完成=2、失败=3

## 一、数据集核心基础信息（代码生成前置知识）

### 1.1 数据集唯一标识

全称：CFDBench。

唯一主键：sample_id，代码内统一作为样本唯一标识进行读取、缓存与结果归档。

数据类型：面向流体代理建模的多问题、多边界条件 CFD 基准数据。

官方或通用数据源：CFDBench 官方仓库与论文说明。

数据规模说明：覆盖四个经典问题：lid-driven cavity、circle tube、dam break、flow around a cylinder；强调边界条件、物性与几何变化下的泛化评估。 代码实现时必须同时记录数据集版本、来源链接、生成日期或下载日期，以便结果可追溯。

### 1.2 任务对象与解析优先级

| 任务对象 | 代码标识 | 解析优先级 | 处理逻辑 |
| --- | --- | --- | --- |
| 方腔驱动流 | CAVITY | 高 | 规则网格与速度压力场 |
| 圆管流 | TUBE | 高 | 关注入口/出口条件和管径变化 |
| 溃坝问题 | DAM | 高 | 重视自由表面或液位演化 |
| 圆柱绕流 | CYLINDER | 高 | 处理几何变化与尾涡特征 |

优先级的含义不是“只读高优先级对象”，而是要求批处理入口优先保证主任务样本完整可用；中低优先级对象通常作为补充信息或容错路径存在，在资源紧张、格式不完整或任务目标仅关注主体字段时可降级处理。

## 六、校验规则与异常处理（判断条件+分支逻辑）

### 6.1 必做校验（代码必须实现，顺序执行）

sample_id 非空且可唯一定位样本，否则直接跳过。

主文件存在且大小大于 0；空文件或损坏文件不得进入正式解析。

核心字段维度一致，例如网格节点数与节点场第一维一致、时序长度一致、输入输出栅格尺寸一致。

数值合法：不允许主字段全部为 NaN/Inf；边界条件与参数字段应可转换为数值或枚举。

样本质量阈值满足最小要求，否则只记录日志，不保存正式结果。

### 6.2 异常处理分支（固定捕获+处理逻辑）

| 异常类型 | 触发场景 | 代码处理逻辑 |
| --- | --- | --- |
| FileNotFoundError | 主文件或元数据缺失 | 记录缺失并跳过该样本 |
| ValueError | 字段维度或数值类型异常 | 标记样本损坏并停止保存 |
| MemoryError | 大样本一次性载入失败 | 切换为分块读取，仅保留核心字段 |
| PermissionError | 目录权限不足 | 自动创建目录并重试一次 |

## 七、数据过滤与清洗规则（剔除条件清单）

### 7.1 数据过滤与清洗规则

过滤空样本：主数组为空、网格为空或轨迹长度为零的样本直接剔除。

过滤维度异常：尺寸与规范不一致、时间维或节点维缺失的样本直接剔除。

过滤数值异常：主场变量出现大量 NaN、Inf 或全零异常时直接剔除。

过滤重复样本：同一主键重复出现时仅保留最新且最完整的一个版本。

过滤低信息量样本：例如有效域比例过低、几何退化、字段全常数等情况只记日志不入库。

清洗动作应在解析过程中实时执行，而不是等全部样本读完后再集中删除。这样可以减少内存占用，也更便于定位是哪一步导致样本失败。

## 八、批量调度与存储规范（路径+命名+格式）

### 8.1 目录规范（代码自动创建，禁止修改路径）

原始文件缓存目录、结果目录、日志目录应彼此分离；结果目录下建议按样本主键首字符或 split 分桶，避免单目录下文件过多。对于大型时序数据，允许只保存摘要 JSON 与索引文件，把大体量 numpy/h5 缓存放到独立 cache 目录。

### 8.2 结果格式规范（固定 JSON 结构）

```json
{
  "sample_id": "cavity_000102",
  "problem_type": "lid_driven_cavity",
  "grid_shape": [128, 128],
  "physics": {"viscosity": 0.01, "bc": "top_lid"},
  "fields": {"u_shape": [128,128], "v_shape": [128,128], "p_shape": [128,128]}
}
```

### 8.3 批量调度规则

批量处理模式：先进先出（FIFO），按传入主键顺序执行。

断点续传：已存在有效结果时跳过重复解析；若只存在失败日志，则允许重试。

执行状态标记：待处理=0、解析中=1、已完成=2、失败=3。

大样本优先输出轻量摘要，避免结果目录中直接堆放超大中间件。

## 九、代码模块化设计（函数拆分规范）

### 9.1 函数拆分规范

load_sample(sample_id: str) -> str：定位单样本容器

read_problem_meta(sample_id: str) -> dict：读取问题类型与物性

save_result(result: dict) -> None：按问题类型保存结果

### 9.2 模块边界要求

各函数必须遵循单一职责原则：定位文件的函数不负责解析内容，解析函数不负责决定存储路径，存储函数不负责再次修改主字段。这样做可以让同一份读取逻辑在训练、评估、可视化和调试四类任务中复用。

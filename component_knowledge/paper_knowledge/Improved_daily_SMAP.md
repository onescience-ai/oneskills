# 维度一：整体模型知识（Model-Level Knowledge）

## 1. 核心任务与需求

### 1.1 核心建模任务
- 任务类型：
  - 利用深度学习和迁移学习，预测 **中国区域 SMAP L4 日尺度表层土壤湿度（SM）**，支持 3 / 5 / 7 天提前量（lead time）的预报。
- 目标：
  - 在 **SMAP 日尺度样本数量不足（< 2300 条）** 的情况下，仍然获得可靠、泛化性良好的 SM 预报；
  - 验证常用水文 DL 模型（CNN、LSTM、ConvLSTM）在小样本 SMAP 数据上的可行性；
  - 利用 **ERA5‑Land 长时间序列 SM 与相关变量** 作为源域，通过迁移学习提高 SMAP 目标域上的预测精度。

### 1.2 主要科学与工程痛点
- SMAP L4 日尺度数据时间跨度短（2015–2020）、重访周期 2–3 天，有效样本 < 2300：
  - 标准 DL 模型通常需要大量样本避免过拟合，小样本下难以直接训练稳定模型；
- 土壤湿度具有强烈的时空变率：
  - 受降水、土壤温度、蒸散、植被、土壤性质等多因子影响，机理复杂；
- 过程型陆面/水文模型（LSM、ESM 等）：
  - 依赖复杂物理机理方程和参数化，计算成本高，人为假设多，难以直接适配小区域小样本任务；
- ERA5‑Land 等重分析产品：
  - 时间长、覆盖广，但与卫星观测存在系统偏差（区域/气候带相关），需要评估其信息是否适合迁移到 SMAP 上。


## 2. 解决方案与整体框架

### 2.1 总体思路
1. 搭建三类典型水文 DL 模型：
   - 纯空间模型：CNN；
   - 纯时间模型：LSTM；
   - 时空耦合模型：ConvLSTM；
   - 先在 **SMAP 目标域** 上，直接用小样本训练三类模型，评估其基线能力。

2. 采用跨源迁移学习（ERA5‑Land → SMAP）：
   - 将 ERA5‑Land 视为源域（时间长、样本多），在其上预训练 CNN / LSTM / ConvLSTM；
   - 将 SMAP 视为目标域（时间短、样本少），在其上对预训练模型进行 **微调（fine‑tuning）**；
   - 对比“直接在 SMAP 训练的 DL 模型”与“迁移 DL 模型”的性能差异。

3. 输入特征与任务设置：
   - 输入因子：
     - 滞后土壤湿度（lagged SM）、降水、土壤温度（以及季节信息）等；
   - 预测目标：
     - SMAP L4 表层（约 0–5 cm）日平均土壤湿度，提前量分别为 3 / 5 / 7 天；
   - 评价指标：
     - 决定系数 \(R^2\) 与均方根误差 RMSE，按中国域及不同气候分区评估。

### 2.2 SM 预测框架（Fig. 2）
- 框架分为两条 pipeline：
  1. **非迁移 DL 模型**（仅用 SMAP）：
     - 在目标域上直接训练 CNN / LSTM / ConvLSTM；
     - 检验小样本下这些模型能否获得足够精度。
  2. **迁移 DL 模型**（ERA5‑Land → SMAP）：
     - 第一步：在 ERA5‑Land 上预训练 CNN / LSTM / ConvLSTM；
     - 第二步：在 SMAP 上微调模型参数；
     - 对比相同结构的“原生模型”和“迁移模型”性能。

- 输出：
  - 给定历史若干天（滞后 SM + 降水 + 土壤温度）的输入序列与空间场，预测未来第 \(t+n\) 天（\(n=3,5,7\)）的 SMAP SM 空间分布。


## 3. 模型与数据架构概览

### 3.1 数据域与划分
- 研究区域：
  - 中国：\(3^\circ\text{N}–54^\circ\text{N},\, 72^\circ\text{E}–126^\circ\text{E}\)。

- 目标域（Target）：SMAP L4
  - 时间：2015‑04‑01 至 2020‑12‑08，日尺度（3 小时产品聚合到日）；
  - 空间：原始 ~0.1°（617×685 格点），因显存限制重采样到 ~1°（52×65 格点）；
  - 变量：
    - 目标：表层（0–5 cm）土壤湿度 SM；
    - 强迫/辅助：降水、土壤温度等（文中仅详细列出关键三因子）。
  - 样本数：约 2080 个日样本；
  - 划分：
    - 训练：2015‑04‑01 至 2018‑12‑07；
    - 验证：2018‑12‑08 至 2019‑12‑07；
    - 测试：2019‑12‑08 至 2020‑12‑08。

- 源域（Source）：ERA5‑Land
  - 时间：1980‑01‑01 至 2020‑12‑08，日尺度；
  - 空间：原始 0.1°，重采样到 1°（52×65，与 SMAP 对齐）；
  - 变量：表层（0–7 cm）土壤湿度、降水、土壤温度等；
  - 样本数：约 14,610（每日一条）；
  - 划分：
    - 训练：1980‑01‑01 至 2019‑12‑07；
    - 验证：2019‑12‑08 至 2020‑12‑08；
    - 无需源域测试集（仅关注其对 SMAP 迁移的效果）。

### 3.2 主要 DL 模型（概览）
- CNN：
  - 适合空间模式提取，输入为多通道 2D 特征图（时间步 × 变量 × 空间）。
- LSTM：
  - 适合时间序列建模，通常对每个格点独立建模时间序列（文中详细结构在后）。
- ConvLSTM：
  - 同时建模时间和空间依赖，捕捉时空特征，对土壤湿度这类时空过程尤为适合。

- 最终结论（文中结果）：
  - 在不迁移时，ConvLSTM 已经优于 CNN 与 LSTM；
  - 迁移学习后，**迁移 ConvLSTM** 提升最大，R² 约 0.909–0.916，RMSE 约 0.0239–0.0247，在 3/5/7 天预报上均表现最佳，一些区域解释方差提升超过 20%。


## 4. 关键公式与评价指标

### 4.1 误差指标
1. RMSE（均方根误差）（公式 (1)）：
\[
\mathrm{RMSE} = \sqrt{\frac{1}{N}\sum_{i=1}^N (y_i - \hat y_i)^2},
\]
其中 \(y_i\) 为真值（土壤湿度），\(\hat y_i\) 为预测值，\(N\) 为样本数。

2. 决定系数 \(R^2\)（公式 (2)）：
\[
R^2 = 1 - \frac{\sum_{i=1}^N (y_i - \hat y_i)^2}{\sum_{i=1}^N (y_i - \bar y)^2},
\]
其中 \(\bar y\) 为真值平均。\(R^2\) 越接近 1，拟合越好。

### 4.2 归一化（公式 (3)）
- 所有变量在格点上进行 min–max 归一化：
\[
 x_{\mathrm{norm}} = \frac{x - x_{\min}}{x_{\min} - x_{\max}},
\]
其中 \(x\)、\(x_{\max}\)、\(x_{\min}\)、\(x_{\mathrm{norm}}\) 分别为原始值、最大值、最小值和归一化值（按训练集统计）。

> 注：公式中分母写作 \(x_{\min} - x_{\max}\) 可能是排版误差，通常应为 \(x_{\max} - x_{\min}\)，这里保持论文写法并作此说明。


## 5. 具体模型与实现细节

### 5.1 CNN 结构（3.1 节）

1. 输入/输出设计
- 输入特征：
  - 变量：SM、降水、土壤温度；
  - 时间步：\(t, t-1, t-2\)，共 3 天；
  - 每个变量每个时间步为 2D 特征图（52×65）；
  - 因此输入张量维度为：\(9\times52\times65\)（通道数 9 = 3 变量 × 3 时间步）。

- 输出：
  - 未来第 \(t+n\) 天的 SMAP SM 场：大小同输入空间：52×65；
  - \(n = 3, 5, 7\) 分别对应 3 / 5 / 7 天 lead time。

2. 网络结构
- 第一卷积层：
  - 通道数：48；
  - 卷积核大小：3×3；
  - padding：0；步长 stride：3；
  - 激活：ReLU；
  - 输出特征图尺寸：48×17×21。

- 后续：
  - 将卷积输出展平后接全连接层（FC），回归输出目标 SM 场；
  - 顶层损失使用欧氏损失（MSE），替换分类常用的 softmax loss。

3. 训练设置
- 迭代轮数：50；
- batch size：128；
- 优化器：Adam；
- 实验观察：更大迭代轮与更小 batch 并未显著提升精度，反而计算成本更高，因此采用上述折中配置。

### 5.2 ConvLSTM（与 LSTM）结构
- 文中对 ConvLSTM 与 LSTM 具体层数/隐藏维度未给出精确数字（记为“未明确说明”），但可以总结其核心特点：
  - LSTM：按格点时间序列独立建模，输入为各因子时间序列，输出为未来某日 SM 标量；
  - ConvLSTM：
    - 以 3D 张量（时间×高度×宽度）为输入，门控结构中的矩阵乘积替换为卷积运算；
    - 能同时捕获时间依赖与空间相关性；
    - 输出未来某天 SM 空间场，长度同 CNN 输出。

- 图 3 显示了 **不使用迁移学习的 ConvLSTM** 在中国域的 bias、RMSE 和 \(R^2\) 空间分布，为之后与“迁移 ConvLSTM”对比提供基准。

### 5.3 迁移学习策略
- 步骤：
  1. 在 ERA5‑Land 上预训练 CNN / LSTM / ConvLSTM：
     - 输入为 ERA5‑Land 的 SM、降水、土壤温度等；
     - 目标为 ERA5‑Land SM；
     - 训练至在源域验证集上收敛（具体 epoch、lr 未在片段中给出）。
  2. 将预训练模型参数迁移到 SMAP 任务：
     - 在 SMAP 训练集上使用较小学习率进行 fine‑tune；
     - 可冻结前几层（特征提取层），仅微调后续层，也可全模型微调（论文片段未明确策略）。

- 关键假设：
  - ERA5‑Land 与 SMAP 在同一气候区域有相似的时空变化模式（Fig. 1c 中显示大多数区域相关系数为正）；
  - ERA5‑Land 更长的时间序列包含丰富的时空特征，可为 SMAP 小样本提供先验结构。


## 6. 实验结论（定性）

- 非迁移 DL 模型：
  - ConvLSTM 在 RMSE 与 \(R^2\) 上优于 CNN 与 LSTM，说明同时建模时空特征有优势；
  - 但在部分区域仍存在较大偏差，表现出小样本下过拟合或信息不足问题。

- 迁移 DL 模型：
  - 所有迁移版本（transfer‑CNN, transfer‑LSTM, transfer‑ConvLSTM）均优于对应非迁移版本；
  - **迁移 ConvLSTM** 效果最好：
    - \(R^2\) 提升至 0.909–0.916；
    - RMSE 降至约 0.0239–0.0247；
    - 回归线 \(y=\hat y\) 与理想线 \(y=x\) 更接近；
    - 在某些区域，解释方差提升 > 20%。

- 因子重要性讨论（文中后续章节）：
  - 分析了不同 SM 值段、降水强度、土壤温度与季节对预测性能的影响；
  - 总体上，滞后 SM 与降水是关键因子，土壤温度与季节信息进一步改善对干湿转换和季节循环的刻画。

- 结论：
  - 对于像 SMAP 这样 **时间短、小样本** 的新建卫星产品，
    - 直接训练 DL 模型可能受限于样本量；
    - 利用 ERA5‑Land 等长序列重分析产品进行跨源迁移，可以显著提高预测精度；
  - 作者建议在类似新建数据集的 SM 预测任务中，优先考虑 **跨源迁移学习 + ConvLSTM 类时空模型**。


# 维度二：基础组件知识（Component-Level Knowledge）

## 组件 1：数据域与预处理模块（SMAP / ERA5‑Land）

1. 子需求定位
- 在 GPU 内存可承受范围内，构建一个代表性又可训练的 SM 预测数据集；
- 同时保证源域与目标域在空间栅格上对齐，便于迁移学习。

2. 技术要点
- 空间降采样：
  - 将原始 0.1°（617×685）重采样到 1°（52×65），采用 `xesmf` Python 包（再网格化）；
  - 参考 Rasp 等工作表明，高分辨率与较粗分辨率在预测性能上差别有限。

- 时间聚合：
  - 将 3 小时 SMAP 产品聚合为日尺度；
  - ERA5‑Land 直接使用日平均或日值。

- 分区统计：
  - 将中国划分为 8 个气候带，分别计算 SMAP / ERA5‑Land SM 的最小值、最大值、均值、中位数、标准差等（表 1）；
  - 用于分析 ERA5‑Land 与 SMAP 的系统偏差模式（干区低估、湿区高估等）。

3. 耦合度
- 强绑定-任务特异：网格大小、时段划分、气候分区完全基于中国 SMAP/ERA5‑Land 特征；
- 直接影响 DL 模型的输入输出尺寸与训练样本量。


## 组件 2：CNN 空间特征提取器

1. 子需求
- 利用周围空间格点信息，提高未来 SM 预测的空间一致性与细节表达。

2. 技术要点
- 3×3 卷积核与 stride=3：在保证感受野的同时，降低中间特征尺寸，减少参数量；
- 通过 48 个通道编码不同组合的空间模式（降水带、土壤湿度梯度等）。

3. 创新性
- 结构本身较为基础，创新主要在 **如何与迁移学习及 ConvLSTM 等模型对比**，而非 CNN 架构本身。


## 组件 3：ConvLSTM 时空建模器

1. 子需求
- 捕捉 SM 在时间与空间上的联合动态：
  - 地表 SM 具有明显的记忆效应（滞后）；
  - 降水、土壤温度等在空间上呈现连续结构；
  - 需要模型同时处理“何时下雨”和“哪里下雨”的信息。

2. 技术要点
- 将传统 LSTM 的矩阵乘法替换为卷积：
  - 输入门、遗忘门、输出门与候选状态都基于卷积运算；
  - 隐状态本身是一个 feature map（H×W×C），随时间滚动；
- 对整个中国区域构建一个统一的 ConvLSTM 模型，而非格点独立训练时间序列。

3. 创新性
- 将 ConvLSTM 用于 SMAP SM 预测的迁移学习场景，首次系统评估其在小样本高维空间场任务中的优势；
- 结果显示：ConvLSTM 是最适合该任务的基干架构，尤其在迁移学习后效果突出。


## 组件 4：跨源迁移学习（ERA5‑Land → SMAP）

1. 子需求
- 解决“小样本 SMAP”条件下 DL 模型易过拟合的问题；
- 利用 ERA5‑Land 长时间序列中蕴含的气候与季节性知识为模型提供先验。

2. 技术与流程
- 流程（概念）：
  1. 在 ERA5‑Land 上训练 CNN/LSTM/ConvLSTM 至收敛；
  2. 将预训练权重复制到目标任务；
  3. 在 SMAP 训练集上微调，使用较小学习率与早停策略避免过拟合。

3. 创新点
- SM 预测领域首个系统使用 **ERA5‑Land → SMAP** 的迁移学习案例之一；
- 证明尽管 ERA5‑Land 与 SMAP 存在系统偏差，但其时空变化模式相似性足以为目标域提供有效先验。


## 组件 5：因子重要性与分情景评估

1. 子需求
- 理解不同环境条件下模型预测能力差异，指导特征选择与后续改进。

2. 技术要点
- 按不同 SM 值段、降水强度、土壤温度、季节等切片评估模型性能：
  - 比较在干旱/湿润区域、不同季节（如雨季/旱季）下 RMSE 与 \(R^2\)；
  - 分析滞后 SM 与降水、温度单独和组合作为输入时的性能差异。

3. 结论性发现（定性）
- 滞后 SM 与降水是最关键的预测因子；
- 土壤温度与季节信息进一步改进了对季节循环和极端事件的描述；
- 迁移 ConvLSTM 在大部分情景下均保持最高的稳定性和精度。


## 组件 6：训练与实验环境

1. 子需求
- 在常规科研硬件环境下完成较大的时空 DL 训练实验。

2. 技术要点
- 硬件：Intel i7‑10750H CPU, GeForce RTX 2060 GPU；
- 软件：PyCharm + PyTorch；
- 通过降采样与简单网络结构控制显存与计算量。

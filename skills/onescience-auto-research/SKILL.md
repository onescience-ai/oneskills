# OneScience 自主研究技能

## 描述

OneScience 自主研究技能是 AI 研究的自主编排层，管理从文献综述、创意生成、实验执行到论文写作的完整研究生命周期。它采用双循环架构，根据需要路由到专业领域技能。

## 核心功能

### 双循环架构

**外循环**：研究生命周期管理
- **文献综述**：搜索相关论文和现有方法
- **创意生成**：基于文献生成新研究想法
- **实验设计**：设计实验方案和基线对比
- **代码生成**：生成训练、评估代码
- **结果分析**：分析实验结果
- **论文写作**：生成论文草稿

**内循环**：专业技能路由
- **模型架构**：路由到模型架构技能
- **分布式训练**：路由到分布式训练技能
- **模型微调**：路由到微调技能
- **模型评估**：路由到评估技能
- **推理优化**：路由到推理技能

### 自动化研究流程

1. **问题定义**：理解研究目标和科学问题
2. **文献调研**：搜索相关论文和现有方法
3. **创意生成**：基于文献生成新研究想法
4. **实验设计**：设计实验方案和基线对比
5. **代码生成**：生成训练、评估代码
6. **结果分析**：分析实验结果
7. **论文写作**：生成论文草稿

## 使用场景

- 自主进行 AI 研究项目
- 快速验证研究想法
- 生成可发表的研究论文
- 复现和改进现有论文

## 技能路由

Autoresearch 会根据任务类型自动路由到以下专业技能：

| 任务类型 | 路由技能 |
|---------|---------|
| 模型架构设计 | Model Architecture |
| 模型微调 | Fine-Tuning |
| 分布式训练 | Distributed Training |
| 模型评估 | Evaluation |
| 推理优化 | Inference |
| 数据处理 | Data Processing |

## 输入参数

### 基础参数

- `research_topic`：研究主题
- `research_objective`：研究目标
- `scientific_question`：科学问题

### 配置参数

- `max_iterations`：最大迭代次数
- `timeout`：超时时间
- `output_format`：输出格式

### 研究目录配置

- `research_dir`：研究目录路径（默认：`./research`）
- `data_dir`：数据目录路径（默认：`./data`）
- `code_dir`：代码目录路径（默认：`./code`）
- `output_dir`：输出目录路径（默认：`./output`）
- `config_file`：配置文件路径（默认：`./config.yaml`）

## 研究目录结构

### 默认目录结构

```
./
├── research/              # 研究目录（默认）
│   ├── literature/        # 文献综述
│   ├── ideas/             # 创意生成
│   ├── experiments/       # 实验设计
│   └── papers/            # 论文写作
├── data/                  # 数据目录（默认）
│   ├── raw/               # 原始数据
│   ├── processed/         # 处理后的数据
│   └── datasets/          # 数据集
├── code/                  # 代码目录（默认）
│   ├── training/          # 训练代码
│   ├── evaluation/        # 评估代码
│   └── inference/         # 推理代码
├── output/                # 输出目录（默认）
│   ├── models/            # 模型
│   ├── results/           # 结果
│   └── logs/              # 日志
├── config.yaml            # 配置文件（默认）
└── README.md              # 研究项目说明
```

### 目录说明

- **research/**：存放所有研究相关的文档和笔记
  - `literature/`：文献综述和调研笔记
  - `ideas/`：创意生成和研究想法
  - `experiments/`：实验设计和结果
  - `papers/`：论文草稿和最终版本

- **data/**：存放所有数据文件
  - `raw/`：原始数据文件
  - `processed/`：处理后的数据
  - `datasets/`：数据集定义和配置

- **code/**：存放所有代码文件
  - `training/`：训练脚本和配置
  - `evaluation/`：评估脚本和指标
  - `inference/`：推理脚本和服务

- **output/**：存放所有输出文件
  - `models/`：训练好的模型
  - `results/`：实验结果和图表
  - `logs/`：训练和运行日志

- **config.yaml**：研究配置文件
  - 数据集配置
  - 模型配置
  - 训练配置
  - 输出配置

## 最佳实践

1. **明确研究目标**：清晰定义科学问题和贡献

3. **可复现实验**：确保实验可复现
4. **完整论文写作**：生成结构完整的论文草稿



### 【第一部分：物理先验注入范式】

#### 1.1 地球几何与 3D 绝对位置嵌入 (3D Absolute Position Embedding) * **【原理描述】**
  * **核心定义**：摒弃传统视觉模型中无物理意义的 1D/2D 相对位置编码，针对气象变量强烈依赖经度、纬度和垂直高度（气压层）的物理现实，构建三维可学习的绝对位置嵌入。
  * **数学推导**：
    分别学习三个维度的嵌入向量：$e_z \in \mathbb{R}^{D/3}$、$e_{lon} \in \mathbb{R}^{D/3}$、$e_{lat} \in \mathbb{R}^{D/3}$。
    拼接为完整的 3D 绝对位置向量：$pos_{3d} = [e_z, e_{lon}, e_{lat}] \in \mathbb{R}^D$。
  * **理论依据**：不同纬度、高度上的大气流体动力学特征迥异（如极地与赤道的科里奥利力差异、高空急流与近地面摩擦），模型必须精确感知每个 Patch 所在的地球三维坐标。
* **【数据规格层】**
  * **输入/输出张量形状**：每个 Patch 对应的物理坐标 $(lon, lat, level)$ 被映射为形状为 $[D]$ 的位置向量。
* **【架构层】**
  * 计算流程图: |
      输入 Patch Token [Batch, N_patches, D]
        ↓
      查找其对应的 (lon, lat, level) 索引
        ↓
      分别查表获取 e_lon, e_lat, e_level -> 形状均为 [Batch, N_patches, D/3]
        ↓
      拼接 (Concat) 得到 pos_3d -> 形状: [Batch, N_patches, D]
        ↓
      与 Patch Token 相加 -> 注入物理位置先验后的 Token [Batch, N_patches, D]
* **【设计层】**
  * **设计动机**：提高位置信息利用效率，弥补标准 ViT/Swin 对地理/垂直位置信息建模不足的缺陷。
  * **创新突破**：使模型真正拥有了“地球坐标系”的空间感知能力，而不是将气象场仅仅当成一张 RGB 图片处理。

#### 1.2 球面均匀性与位置加权损失 (Position-weighted Loss)
* **【原理描述】**
  * **核心定义**：在损失函数中结合可学习的变量绝对位置权重 $f(v)$ 与固定的标准纬度权重 $L(i)$，对特定纬带或变量区域施加差异化惩罚。
  * **数学推导**：
    纬度权重：$$L(i) = \frac{\cos(\mathrm{lat}(i))}{\frac{1}{H}\sum_{i'=1}^H \cos(\mathrm{lat}(i'))}$$。
    加权损失：$$\mathcal{L} = \frac{1}{CHW} \sum_{c=1}^C\sum_{i=1}^H\sum_{j=1}^W f(v) L(i) \big(\hat{X}_{i+\Delta t}^{cij} - X_{i+\Delta t}^{cij}\big)^2$$。
* **【设计层】**
  * **设计动机**：某些纬带或特定位置上的误差对人类社会影响更大（如中纬度人口密集区），通过 $f(v)$ 在物理关键区域提升精度优先级。

### 【第二部分：变量组织范式】

#### 2.1 2D通道堆叠与跨变量注意力聚合策略
* **【原理描述】**
  * **核心定义**：将 $C=20$ 个分布在多气压层和地表的气象变量视为通道堆叠输入。随后借鉴 ClimaX 思路，使用 Cross-Attention 机制在通道间聚合信息，提取跨变量的物理耦合关系。
* **【数据规格层】**
  * **输入张量形状**：$X \in \mathbb{R}^{C \times H \times W}$，其中 $C=20$，$H=721$，$W=1440$（对应 0.25° 分辨率）。Patch 切分大小为 $p=8$。
* **【架构层】**
  * 计算流程图: |
      多通道气象场 [B, C, H, W]
        ↓
      切分为 8x8 Patch 并线性嵌入 -> 形状: [B, C, (H/8)*(W/8), D_patch]
        ↓
      可学习 Query 与各通道 Token 执行 Cross-Attention 聚合
        ↓
      输出融合后的统一空间 Token 序列 -> 形状: [B, (H/8)*(W/8), D]

### 【第三部分：主干网络类型范式】

#### 3.1 基于 Transformer 系列 (ViT Encoder-Decoder)
* **【原理描述】**
  * **核心定义**：模型主干采用编码器-解码器架构。编码器由 6 层维度为 768 的 Transformer Block 构成；解码器由 6 层维度为 512 的 Transformer Block 构成。

#### 3.2 稀疏混合专家架构范式 (Sparse MoE) * **【原理描述】**
  * **核心定义**：在 Encoder 的每一层中，将标准的全连接前馈网络（FFN）替换为由 $N$ 个独立 FFN 组成的专家库，通过门控网络（Gating Network）动态将 Token 路由给最相关的少数几个专家处理。
  * **数学推导**：
    门控网络 logits：$xW + \epsilon$，其中 $\epsilon \sim \mathcal{N}(0, 1/e^2)$。
    Top-k 路由 (k=2)：
    $$g(x) = \mathrm{Softmax}(\mathrm{Top\!-k}(xW + \epsilon, k))$$。
    MoE 输出：
    $$y = \sum_{i=1}^N g_i(x) E_i(x)$$。
  * **理论依据**：天气系统包含从热带气旋到极地涡旋等极其复杂的物理模态。MoE 允许模型解耦这些模态，让不同的“专家”专精于特定的气候特征，从而在不增加单次推理计算量的前提下，成倍扩大模型的参数容量。
* **【数据规格层】**
  * **超参数**：专家总数 $N=20$，路由激活数 $k=2$。
* **【架构层】**
  * 计算流程图: |
      输入 Token x [Batch*SeqLen, D]
        ↓
      乘以路由权重 W 并注入高斯噪声 epsilon -> Logits [Batch*SeqLen, N=20]
        ↓
      Top-2 掩码 (非 Top-2 置为负无穷) 并 Softmax -> 路由门控权重 g(x) [Batch*SeqLen, 20]
        ↓
      按权重分发给被激活的专家 FFN_i 进行前向计算 -> E_i(x)
        ↓
      按 g(x) 加权求和 -> 输出 Token y [Batch*SeqLen, D]
* **【对比层】**
  * **痛点解决**：现有的 Pangu/GraphCast 依赖密集网络，参数扩展会带来指数级算力开销。MoE 成功打破了这一瓶颈，使得用有限 GPU 资源挑战 SOTA 成为可能。

### 【第五部分：训练与优化范式】

#### 5.1 自回归多步推演范式 (Auto-Regressive Rollout) * **【原理描述】**
  * **核心定义**：模型仅训练单步 $6$ 小时的状态映射算子，在推理时将输出作为下一步的输入进行自回归滚动，最长演化至 8 天（32 步）。
  * **数学推导**：
    $$\hat{X}_{i+1} = f_\theta(X_i), \quad \hat{X}_{i+2} = f_\theta(\hat{X}_{i+1}), \dots$$。

#### 5.2 MoE 负载均衡优化范式 (Load-balancing Auxiliary Loss)
* **【原理描述】**
  * **核心定义**：为了防止模型在训练中发生“路由坍塌”（即所有 Token 都涌向少数几个专家，导致其他专家闲置），引入单一且可微的负载均衡辅助损失。
  * **数学推导**：
    辅助损失：$$l_{aux} = E \sum_{i=1}^E h_i P_i$$。
    其中，Token 分配频率 $h_i = \frac{1}{L} \sum_{x \in B} \mathbf{1}\{\arg\max g(x) = i\}$。
    路由概率质量 $P_i = \frac{1}{L} \sum_{x \in B} g_i(x)$。
  * **设计动机**：通过最小化 $l_{aux}$，强制驱动 $h_i$ 与 $P_i$ 接近均匀分布，最大化专家库的利用率和模型的泛化能力。

#### 5.3 极低资源数据高效训练策略
* **【原理描述】**
  * **设计突破**：EWMoE 彻底抛弃了气象大模型依赖三十年以上 ERA5 数据的“暴力美学”。它仅使用了 **2 年（2015-2016）** 作为训练集，在有限算力下全面超越了同样设定下的 FourCastNet 和 ClimaX，并逼近 Pangu-Weather 等头部模型。

### 【第六部分：评估与验证范式】

#### 6.1 评估指标范式
* **【原理描述】**
  * **核心定义**：使用标准气象学指标量化模型预报场与 ERA5 分析场之间的偏差和相位一致性。
  * **核心指标**：
    1. **纬度加权 ACC (Anomaly Correlation Coefficient)**：
       $$\mathrm{ACC}(v,l) = \frac{\sum_m L(m)\,\hat{X}_{pred}\,\hat{X}_{true}}{\sqrt{\sum_m L(m)\hat{X}_{pred}^2\,\sum_m L(m)\hat{X}_{true}^2}}$$
       用于评估模型去气候平均后的异常场相关性。
    2. **纬度加权 RMSE**：
       $$\mathrm{RMSE}(v,l) = \sqrt{\frac{1}{NM}\sum_m^M\sum_n^N L(m)(X_{pred}-X_{true})^2}$$
       评估物理场的绝对误差量级。

#### 6.2 下游任务验证范式
* **【原理描述】**
  * **验证场景**：重点评估了模型在 1-8 天中期预报（Medium-range forecasting）下的关键大气变量演变，特别是 500 hPa 位势高度 (Z500)、850 hPa 温度 (T850)、2m 温度 (T2m) 和 10m 纬向风 (U10)。
  * **评估结果**：在 1-3 天时效上精度与 Pangu-Weather、GraphCast 相当；在较长时效推演中，由于 MoE 架构的高容量和稳定性，其退化速度明显更慢。

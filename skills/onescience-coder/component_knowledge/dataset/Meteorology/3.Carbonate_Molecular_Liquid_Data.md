## 一、数据集核心基础信息（代码生成前置知识）

### 1.1 数据集唯一标识

全称：碳酸盐类分子液体数据集（以 EC:EMC 二元溶剂训练/测试集为代表）

唯一主键：sample_id + frame_id（组合键）

数据类型：分子液体结构、能量、力、体积扫描与分子动力学轨迹

官方数据源：Zenodo 数据集：Machine Learning Force Field for Molecular Liquids: EC/EMC Binary Solvent

数据规模：包含训练集、测试集、体积扫描、最终 GAP 势和分子动力学轨迹文件。

补充必要项：需记录溶剂组成、温度、密度、势函数版本与轨迹步长。

### 1.2 数据对象与解析优先级

| 数据对象 | 代码标识 | 解析优先级 | 处理逻辑 |
| --- | --- | --- | --- |
| 训练样本 | TRAIN | 高 | 默认全量解析结构、能量与力 |
| 测试样本 | TEST | 高 | 用于验收，不回写标签 |
| 体积扫描样本 | VOLUME_SCAN | 中 | 保留体积与能量映射 |
| MD 轨迹样本 | TRAJECTORY | 中 | 按帧顺序读取，禁止乱序 |

## 二、数据格式规范与字段映射（必解析字段清单）

### 2.1 主格式

主格式：扩展 XYZ / ASE 可读结构文件（.xyz / .extxyz）—— 代码优先解析。

| 字段名 | 代码变量名 | 数据类型 | 是否必填 | 缺失处理 |
| --- | --- | --- | --- | --- |
| symbols | elements | 数组 | 是 | 终止解析 |
| positions | positions | 数组 | 是 | 剔除当前帧 |
| energy | energy | 浮点数 | 否 | 测试帧可为空 |
| forces | forces | 数组 | 否 | 缺失则填空 |
| cell | cell | 数组 | 否 | 非周期样本可为空 |

### 2.2 配套格式

配套格式：XYZ / LAMMPS 轨迹 / NumPy 数组。按帧顺序解析。

| 行标识/字段 | 字段含义 | 代码变量名 |
| --- | --- | --- |
| frame header | 帧标识与注释 | frame_meta |
| atom lines | 原子符号与坐标 | atoms |

### 2.3 兼容/辅助格式

兼容/辅助格式：Jupyter Notebook、HTML/PDF 说明文档、模型文件。仅记录版本与路径。

## 五、解析逻辑与遍历规则（分层执行步骤）

### 5.1 单条目解析步骤（智能体按顺序生成代码）

定位 XYZ/轨迹文件 → 逐帧解析 atoms、positions、energy、forces → 生成标准结果 → 保存 JSON。

### 5.2 多对象处理规则

同一样本的轨迹帧必须保序；不同温度/配比样本禁止合并。

## 六、校验规则与异常处理（判断条件+分支逻辑）

### 6.1 必做校验（代码必须实现，顺序执行）

sample_id 非空

positions 与 elements 数量一致

forces 维度应为 [n_atoms,3] 或为空

体积与能量应为有限值

### 6.2 异常处理分支（固定捕获+处理逻辑）

| 异常类型 | 触发场景 | 代码处理逻辑 |
| --- | --- | --- |
| ValueError | XYZ 帧损坏或列数异常 | 跳过当前帧 |
| FileNotFoundError | 轨迹文件缺失 | 记录失败并跳过 |
| OSError | 模型文件或轨迹无法打开 | 跳过当前文件 |
| MemoryError | 长轨迹内存溢出 | 改为逐帧流式读取 |

## 七、数据过滤与清洗规则（剔除条件清单）

过滤空帧

过滤原子数与配方不一致的样本

过滤 energy/forces 均缺失且非测试集的样本

过滤重复 frame_id

## 八、批量调度与存储规范（路径+命名+格式）

### 8.1 目录规范（代码自动创建，禁止修改路径）

原始文件缓存：./carbonate_liquid/train/{sample_id}.xyz、./carbonate_liquid/traj/{sample_id}.xyz；解析结果：./storage/{首字母}/{sample_id}_result.json；日志：./logs/{YYYYMMDD}.log。

### 8.2 结果格式规范（固定JSON结构）

```json
{
  "sample_id": "ECEMC_001",
  "frame_id": 0,
  "composition": "EC:EMC",
  "n_atoms": 96,
  "energy": -1250.42
}
```

### 8.3 批量调度规则

FIFO 顺序执行

同一样本轨迹按 frame_id 升序执行

执行状态：待处理=0、解析中=1、已完成=2、失败=3

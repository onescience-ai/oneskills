## 一、数据集核心基础信息（代码生成前置知识）

### 1.1 数据集唯一标识
- 全称：AirfRANS
- 唯一主键：`simulation_id`
- 数据类型：二维翼型周围不可压缩稳态 RANS 数值仿真数据，包含节点、网格、边界标签、速度和压力监督量
- 官方数据源：AirfRANS 官方文档与配套 Python 库
- 数据规模：官方说明包含 1000 个仿真样本，Re 取值在 `2e6` 到 `6e6` 之间，攻角在 `-5°` 到 `15°` 之间，并定义 4 个机器学习任务

### 1.2 任务对象与解析优先级

| 任务对象 | 代码标识 | 解析优先级 | 处理逻辑 |
|---|---|---|---|
| 全场预测 | FULL_FIELD | 高 | 读取节点、网格和全场速度压力监督，是最常见任务 |
| 表面任务 | SURFACE | 高 | 只保留翼型表面节点及其表面压力等量 |
| 系数回归 | COEFF | 中 | 读取升阻力等标量结果，供快速代理模型训练 |
| 元信息统计 | META | 中 | 读取 Re、AoA、翼型系列等条件参数 |

## 二、数据格式规范与字段映射（必解析字段清单）

### 2.1 主格式与核心字段

| 格式类型 | 字段含义 | 代码变量名 | 数据类型 | 是否必填 | 缺失处理 |
|---|---|---|---|---|---|
| 样本对象 | 坐标、网格、字段和条件参数 | `sample` | 对象/字典 | 是 | 接口报错时重试 |

### 2.3 数据组织说明
- 工程上最稳妥的方式是使用官方 `airfrans` 库直接加载，而不是手工猜测底层序列化结构。
- 如果需要神经算子或图网络训练，可在官方返回对象基础上再转换为图结构或规则张量。
- 全场预测和表面预测的字段集合不同，代码中必须显式记录 `task_type`，避免后续混用。

## 五、解析逻辑与遍历规则（分层执行步骤）

### 6.1 必做校验（代码必须实现，顺序执行）

- Re 和 AoA 应落在官方说明的数值范围内。
- 网格单元数、节点数和边界标签长度应匹配。

### 6.2 异常处理分支（固定捕获+处理逻辑）

| 异常类型 | 触发场景 | 代码处理逻辑 |
|---|---|---|

## 七、数据过滤与清洗规则（剔除条件清单）
代码必须在解析过程中实时过滤，禁止等全部数据读取完成后再统一做清洗。这样做可以减少内存占用，也能更快发现输入源中的结构性问题。

- 过滤缺失核心字段或边界标签的坏样本。
- 按任务要求可过滤非表面节点，只保留翼型表面监督区域。
- 可按 Re 或 AoA 区间构造子集，用于研究参数外泛化。

## 八、批量调度与存储规范（路径+命名+格式）

### 8.1 目录规范（代码自动创建，禁止修改路径）
- 建议一仿真样本一个结果文件，便于图网络或神经算子直接读取。
- 额外导出一份条件索引表，记录 `simulation_id` 与 Re、AoA、`task_type` 的对应关系。
- 原始数据和派生张量应分目录保存，防止训练前误读不同表示。

### 8.2 结果格式规范（固定JSON结构）

```json
{
  "dataset_name": "AirfRANS",
  "sample_id": "simulation_id",
  "meta": {"split": "train", "task": "full", "status": "ok"},
  "payload": {
    "num_nodes": 14235,
    "condition": {"Re": 3000000, "AoA": 4.0},
    "field_names": ["pressure", "velocity"]
  }
}
```

### 8.3 批量调度规则
- 批量处理按 `simulation_id` 顺序执行。
- 坏样本单独写入 `bad_cases` 日志，不允许因单个样本中断全局。
- 可视化仅在 QA 阶段对抽样样本启用，主流程不默认绘图。

## 一、数据集核心基础信息（代码生成前置知识）

### 1.1 数据集唯一标识

全称：AlphaFold v2.3.0 推理数据集 / 运行目录结构。

唯一主键：job_name 或 fasta 文件对应的样本ID；代码内统一转为大写并用于输出目录命名。

数据类型：蛋白单体与蛋白复合物结构预测输入、特征缓存、模板搜索结果与模型输出结果。

官方数据源：Google DeepMind AlphaFold 开源仓库 v2.3.0 代码、模型参数与依赖数据库下载脚本。

数据规模：完整数据库常见部署规模约 2.2TB（full_dbs）；reduced_dbs 模式约 600GB 级，代码需适配大文件、批量任务与共享存储。

版本补充：v2.3.0 新增更新的 AlphaFold-Multimer 权重，训练截止日期更新为 2021-09-30。

### 1.2 运行模式与解析优先级

| 运行模式 | 代码标识 | 解析优先级 | 处理逻辑 |
| --- | --- | --- | --- |
| 单体常规模型 | monomer | 高 | 默认全量解析，单序列输入，输出5个模型结果 |
| 单体 pTM 模型 | monomer_ptm | 高 | 保留 pTM / PAE 相关输出，用于下游排序 |
| CASP14 单体模型 | monomer_casp14 | 中 | 与官方评测配置一致，通常作对照运行 |
| 多聚体模型 | multimer | 高 | 多链联合建模，需额外 UniProt 与 pdb_seqres 数据 |

输入对象范围：默认支持蛋白序列；单体模式输入为1条蛋白序列，多聚体模式输入为同一 FASTA 文件中的多条蛋白序列。AlphaFold2.3.0 不直接原生建模小分子配体。

## 二、数据格式规范与字段映射（必解析字段清单）

### 2.1 主输入格式：FASTA（.fasta / .fa）—— 代码优先解析

格式类型：纯文本序列格式，以 > 开头标识序列头；代码按“序列头 + 氨基酸序列”二元结构解析，禁止按固定字符位切片。

| 行标识 | 字段含义 | 代码提取变量名 | 数据类型 | 是否必填 | 缺失处理 |
| --- | --- | --- | --- | --- | --- |
| > | 样本ID / 链ID | seq_id | 字符串 | 是 | 终止解析，标记无效输入 |
| 序列行 | 氨基酸序列 | sequence | 字符串 | 是 | 终止解析，不进入推理 |
| 多条记录 | 多聚体链集合 | chain_sequences | 列表 | multimer时是 | 链数不足时标记输入不合法 |

### 2.2 模板与结构格式：mmCIF（.cif）

格式类型：PDBx/mmCIF；在 AlphaFold2.3.0 中主要用于模板数据库（pdb_mmcif）与模板命中结构读取，代码按类别 + 关键字提取字段。

| 类别关键字 | 字段名 | 代码变量名 | 数据类型 | 是否必填 | 缺失处理 |
| --- | --- | --- | --- | --- | --- |
| _entry | id | template_id | 字符串 | 是 | 终止当前模板解析 |
| _atom_site | Cartn_x/y/z | coord | 数组[float,float,float] | 是 | 剔除当前原子 |
| _struct_asym | id | chain_id | 字符串 | 是 | 自动生成备用链ID |
| _exptl | method | expt_method | 字符串 | 否 | 缺失则记为 UNKNOWN |

### 2.3 结果与中间文件格式（.pdb / .pkl / .json）

AlphaFold2.3.0 运行目录中的核心输出并非单一格式，而是 PDB 结果文件、pickle 中间结果文件与 JSON 排序/计时文件的组合。

| 文件类型 | 典型文件名 | 核心用途 |
| --- | --- | --- |
| PDB | ranked_0.pdb / relaxed_model_*.pdb | 最终结构输出与松弛后结构输出 |
| PKL | result_model_*.pkl | 保存 pLDDT、PAE、ranking_confidence 等中间结果 |
| JSON | ranking_debug.json / timings.json | 保存模型排序与阶段耗时信息 |
| A3M / STO | MSA缓存结果 | 序列搜索阶段的中间文件 |

## 五、解析逻辑与遍历规则（分层执行步骤）

固定执行顺序：FASTA 输入 → MSA 搜索 → 模板搜索 → 特征构建 → 模型推理 → 排序与松弛 → 结果落盘；禁止跳过基础输入校验。

### 5.1 单条目解析步骤（智能体按顺序生成代码）

初始化输入：读取 FASTA 文件并拆分序列头与氨基酸序列。

构建特征：根据 model_preset 触发单体或多聚体数据管线。

序列搜索：调用 jackhmmer / hhblits / hmmsearch 等工具生成 MSA 与模板候选。

模型推理：运行 5 个模型参数集，输出 result_model_*.pkl。

结果排序：按 ranking_confidence 或 pTM / ipTM 组合得分排序，生成 ranked_*.pdb。

结果封装：将关键结构、置信度与日志写入标准化 JSON 结果。

### 5.2 多链处理规则（Multimer 专属）

判断 FASTA 中链数 > 1 时启用 multimer 流程；所有链按输入顺序记录，禁止在解析结果中打乱链顺序；不同链的序列长度、MSA 与模板命中需保留链级关系。

## 六、校验规则与异常处理（判断条件+分支逻辑）

### 6.1 必做校验（代码必须实现，顺序执行）

样本ID校验：不能为空、不可重复，且应与输出目录一一对应。

FASTA合法性校验：首行必须以 > 开头；序列仅允许标准氨基酸字符。

日期校验：max_template_date 必须满足 YYYY-MM-DD 格式。

结果校验：ranked_0.pdb、ranking_debug.json、timings.json 至少应存在其一；缺失时标记任务失败。

数值校验：pLDDT ∈ [0,100]，PAE ≥ 0；异常值统一记入错误日志。

### 6.2 异常处理分支（固定捕获+处理逻辑）

| 异常类型 | 触发场景 | 代码处理逻辑 |
| --- | --- | --- |
| FileNotFoundError | 输入 FASTA 或数据库路径不存在 | 自动创建输出目录；输入/数据库缺失则终止当前任务 |
| ValueError | FASTA 非法字符或日期格式错误 | 标记输入不合法并跳过 |
| MemoryError | 大规模 MSA / multimer 内存溢出 | 降低并发或切换 reduced_dbs |
| RuntimeError | GPU 推理失败或 JAX 初始化失败 | 记录错误并回退 CPU / 终止当前任务 |

## 七、数据过滤与清洗规则（剔除条件清单）

过滤空序列：长度为0的输入记录直接跳过。

过滤非法字符：非标准氨基酸字符默认判定为无效输入。

过滤残缺输出：仅有临时目录、无 ranked_*.pdb 与 result_model_*.pkl 的任务不保存到正式结果集。

过滤低置信度结果：默认平均 pLDDT < 50 的样本仅记录日志，不作为正式结果输出。

执行状态标记：待处理=0、MSA中=1、推理中=2、已完成=3、失败=4。

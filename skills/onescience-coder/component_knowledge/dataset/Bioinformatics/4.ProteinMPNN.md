## 一、数据集核心基础信息（代码生成前置知识）

### 1.1 数据集唯一标识

全称：ProteinMPNN

唯一主键：Target ID（统一使用样本ID或PDB文件名去扩展名后的字符串标识，代码内统一转为大写）

数据类型：基于蛋白质三维骨架的序列设计输入/输出数据

官方数据源：ProteinMPNN 官方仓库输入样例、helper_scripts 解析结果、用户提供的 PDB/JSONL/FASTA 文件

数据规模：官方仓库以单样本 PDB 和批量 JSONL 为主，实际生产环境可扩展到大规模骨架批处理；代码需适配单文件、目录批处理与 GPU 推理任务。

补充必要项：记录模型权重版本（如 v_48_020）、是否启用 soluble model、是否启用 CA-only 模式、git_hash 与 seed，便于结果复现。

### 1.2 推理对象与解析优先级

| 推理对象 | 代码标识 | 解析优先级 | 处理逻辑 |
| --- | --- | --- | --- |
| 单链蛋白骨架 | MONOMER | 高 | 默认全量解析，生成单条或多条候选序列 |
| 多链复合物骨架 | MULTI_CHAIN | 高 | 按链关系解析，可指定部分链设计、部分链固定 |
| CA-only 骨架 | CA_ONLY | 中 | 启用 --ca_only 并加载 CA-only 权重 |
| 可溶蛋白专用模式 | SOLUBLE_MODEL | 中 | 启用 --use_soluble_model 并使用 soluble_model_weights |
| 评分/概率输出任务 | SCORING_ONLY | 低 | 仅输出 score、conditional probs 或 unconditional probs，不生成常规设计结果 |

输入对象范围：默认支持单链与多链蛋白 PDB 骨架；如需固定位置、链选择、PSSM、氨基酸偏置或 tied positions，必须通过对应 JSONL 控制文件显式声明。

## 二、数据格式规范与字段映射（必解析字段清单）

### 2.1 主格式：PDB（.pdb）—— 代码优先解析

格式类型：固定行宽文本结构格式；ProteinMPNN 默认直接接受 PDB 骨架，或先通过 helper_scripts 解析为 JSONL。代码按原子记录、链ID、残基编号与坐标字段提取，禁止仅按自由文本搜索解析。

| 行标识 | 字段含义 | 代码变量名 | 数据类型 | 是否必填 | 缺失处理 |
| --- | --- | --- | --- | --- | --- |
| ATOM | 链ID | chain_id | 字符串 | 是 | 缺失则标记为无效链 |
| ATOM | 残基编号 | res_id | 整数 | 是 | 缺失则跳过当前残基 |
| ATOM | 残基名称 | resname | 字符串 | 是 | 缺失则跳过当前残基 |
| ATOM | N/CA/C/O 坐标 | backbone_coord | 数组[float,float,float] | 是 | 缺失主链原子则标记骨架不完整 |
| ATOM | B因子/占位字段 | b_factor | 浮点数 | 否 | 默认保留，不作为核心解析依据 |
| TER/END | 结构结束 | end_flag | 字符串 | 否 | 用于终止解析 |

### 2.2 配套格式：JSONL（.jsonl）—— 批量设计核心中间格式

格式类型：逐行 JSON 结构；ProteinMPNN 可通过 helper_scripts 将 PDB 目录预解析为 JSONL，随后由主程序批量读取。

| 字段名 | 代码变量名 | 数据类型 | 是否必填 | 说明 |
| --- | --- | --- | --- | --- |
| name | target_id | 字符串 | 是 | 样本唯一标识 |
| seq | native_seq | 字符串 | 是 | 原始参考序列 |
| num_of_chains | num_chains | 整数 | 是 | 链数量 |
| chain_id_dict / chain_list | chain_meta | 字典/列表 | 否 | 链级设计或固定信息 |
| coords_chain_* | coords | 字典 | 是 | 每条链的骨架坐标集合 |

### 2.3 配套格式：FASTA / JSONL 控制文件（.fasta / .jsonl）

FASTA 用于 path_to_fasta 打分输入序列；JSONL 控制文件用于 chain_id_jsonl、fixed_positions_jsonl、bias_AA_jsonl、omit_AA_jsonl、pssm_jsonl、tied_positions_jsonl 等约束输入。控制文件默认不做全量语义解析，仅做路径存在性、JSON 结构和键字段校验。

补充必要项：输出 FASTA 头部需保留 score、global_score、designed_chains、fixed_chains、model_name、git_hash、seed、temperature、sample 编号等元信息，便于后续筛选与复现。

## 五、解析逻辑与遍历规则（分层执行步骤）

### 5.1 单条目解析步骤（智能体按顺序生成代码）

初始化输入：判断使用 pdb_path 单文件模式还是 jsonl_path 批量模式。

加载骨架：从 PDB 提取链、残基、N/CA/C/O 主链坐标；或直接读取已解析 JSONL。

加载控制文件：按需读取 chain_id_jsonl、fixed_positions_jsonl、bias_AA_jsonl、omit_AA_jsonl、pssm_jsonl、tied_positions_jsonl。

加载模型：根据 ca_only、use_soluble_model 和 model_name 选择正确权重。

执行推理：按 num_seq_per_target、batch_size、sampling_temp 生成序列，或输出分数/概率矩阵。

封装结果：将 FASTA 头部元信息、序列、score/global_score 以及可选 NPY 输出统一归档。

### 5.2 多链处理规则（复合物专属）

判断链数量＞1时，必须保留 designed_chains 与 fixed_chains 的链级关系；禁止合并不同链的设计结果。多链序列使用 “/” 按字母序分隔。

## 六、校验规则与异常处理（判断条件+分支逻辑）

### 6.1 必做校验（代码必须实现，顺序执行）

Target ID 校验：不能为空，且可唯一映射到输入文件。

PDB 完整性校验：至少包含有效链、残基编号与主链原子坐标。

JSONL 合法性校验：每行必须为合法 JSON，且包含 name、seq、coords 等关键字段。

控制文件校验：fixed/tied/bias/pssm 等 JSONL 文件路径存在且键值结构正确。

输出合法性校验：生成 FASTA 时必须包含序列正文；save_score/save_probs 开启时必须生成对应 NPY 文件。

### 6.2 异常处理分支（固定捕获+处理逻辑）

| 异常类型 | 触发场景 | 代码处理逻辑 |
| --- | --- | --- |
| FileNotFoundError | PDB/JSONL/控制文件路径不存在 | 自动记录缺失路径并跳过当前任务 |
| JSONDecodeError | JSONL 控制文件格式错误 | 终止当前样本并标记配置非法 |
| RuntimeError | GPU 显存不足或 Torch 执行失败 | 降低 batch_size 或切换 CPU 后重试一次 |
| ValueError | 参数冲突或字段缺失 | 记录错误原因，不中断整体批处理 |
| MemoryError | 大规模批量处理内存溢出 | 分批加载 JSONL，仅保留当前批次样本 |

## 七、数据过滤与清洗规则（剔除条件清单）

过滤空链：无有效残基坐标的链直接跳过。

过滤不完整骨架：缺失核心主链原子（N/CA/C/O）的残基按无效残基处理。

过滤非法氨基酸约束：omit_AAs、bias_AA、pssm、tied_positions 中存在非法氨基酸字符或非法位置编号时跳过该样本。

过滤异常输出：生成序列长度与目标链长度不一致时，不保存结果文件。

过滤重复结果：同一 target_id、同一 temperature、同一 sample 编号的重复输出不重复保存。

## 八、批量调度与存储规范（路径+命名+格式）

### 8.1 目录规范（代码自动创建，禁止修改路径）

原始输入缓存：./inputs/pdbs/{target_id}.pdb 或 ./inputs/parsed_pdbs.jsonl

解析结果存储：./outputs/seqs/{target_id}.fa、./outputs/scores/{target_id}.npy、./outputs/probs/{target_id}.npy

日志文件：./logs/{YYYYMMDD}.log（按日期归档）

### 8.2 结果格式规范（固定JSON结构）

```json
{
"target_id": "字符串",
"model_name": "v_48_020",
"designed_chains": ["A"],
"fixed_chains": ["B"],
"sampling_temp": 0.1,
"samples": [
{
"sample_id": 1,
"score": 0.7291,
"global_score": 0.9330,
"sequence": "ACDE.../FGHI..."
}
]
}
```

### 8.3 批量调度规则

批量处理模式：先进先出（FIFO），按传入 PDB/JSONL 样本顺序执行。

断点续传：已存在有效 FASTA/NPY 结果时跳过重复设计。

执行状态标记：待处理=0、解析中=1、已完成=2、失败=3。

## 十、实用执行案例（智能体复刻场景）

### 10.1 单条样本解析案例（基础场景）

目标样本：单链蛋白 PDB；执行流程：读取 backbone → 加载 v_48_020 → 生成 1 条序列 → 输出 FASTA。

### 10.2 多链样本处理案例（生产场景）

输入多链复合物 PDB，使用 chain_id_jsonl 指定设计链与固定链，输出多链分隔序列与 score。

通过 fixed_positions_jsonl、tied_positions_jsonl、bias_AA_jsonl 和 pssm_jsonl 组合约束，生成满足设计要求的候选序列。

## 十一、代码测试与验证标准（验收规则）

## 十二、常见报错修复方案（兜底逻辑）

| 报错 | 处理逻辑 |
| --- | --- |
| ModuleNotFoundError | 执行 pip 安装 torch、numpy、biopython、tqdm |
| CUDA out of memory | 减小 batch_size、num_seq_per_target，必要时切换 CPU |
| JSONDecodeError | 检查 JSONL 控制文件是否为逐行 JSON，修复非法逗号或引号 |
| KeyError | 核对控制文件中的链ID、位置编号与目标样本是否一致 |
| 无结果生成 | 检查 out_folder、模型权重路径、score_only/save_probs 等参数是否冲突 |

# 气象AI模型组合适配器设计通用Skill

## 1. 技能目标
当用户需要将不同气象AI模型（如Fuxi、Pangu、FourCastNet、GraphCast等）的部分模块进行组合时，本Skill提供一套标准化的维度适配处理流程。核心目标是：
- **维度解构**：识别各模型对输入/输出数据的变量集合、陆地变量、大气变量、时间维度及特征结构的差异。
- **适配层设计**：通过构建独立、可插拔的适配器模块，实现异构组件间的无缝连接。
- **保形与保真**：确保维度转换的准确性、数值稳定性、可微性（如需训练）及计算效率。

## 2. 适用场景
- **异构编码器-适配器-解码器组合**：如使用Fuxi的编码器 + 维度适配器 + pangu的解码器。
- **模块替换**：将某一模型的特定模块（如FourCastNet的AFNO层）替换为另一模型的对应模块（如Pangu的3D Swin Transformer）。
- **多模型特征融合**：结合不同模型的特征提取方式，如Fuxi的变量嵌入与Pangu的时空位置编码。
- **预训练模型微调与拼接**：利用多个预训练模型的部分权重，构建新的混合架构。

## 3. 通用维度适配框架

### 3.1 维度要素分析
在组合前，需对每个模块的输入/输出进行结构化解析。虽然不同模型的张量顺序各异，但通常包含以下五个核心维度：

| 维度类型 | 描述 | 常见差异示例 |
| :--- | :--- | :--- |
| **批次维度 (B)** | 输入进模型(模块)的样本数 | 传递过程通常保持一致，个别情况下会和第二维度整合，需确保传递时不变。 |
| **时间维度 (T)** | 时间步长、历史帧数、预测步数 | 单步自回归 (T=1) vs. 多帧并行预测 (T>1)；滑动窗口长度差异。 |
| **变量维度 (V)** | 气象变量的种类与数量 | Fuxi使用约70个变量，包含5个陆地变量+5个大气变量x13层气压层；Pangu共69个变量，包含4个陆地变量+5个大气变量*13层气压层；FourCastNet使用20个变量，包含6个陆地变量，将大气变量视为陆地变量。 |
| **空间维度 (H, W)** | 网格分辨率、网格类型（规则/高斯） | FourCastNet: 规则经纬网格 (720, 1440)；Pangu/Fuxi: 高斯网格 (721, 1440)；GraphCast: 均匀经纬网格 (721, 1440)。 |

**重要提示**：模型的张量布局为  `(B, T, V, H, W)` ，可能在模型内部会拆分为`(B, T, V', L, H, W)`等，L代表气压层。适配器的首要任务是识别并统一这种**维度顺序**。

### 3.2 通用适配步骤

#### 步骤1：接口参数提取
对每个待组合的模块，明确其：
- **输入规格**：完整的维度顺序、各维度大小及物理含义。
- **输出规格**：同上。
- **特征结构**：是否使用位置编码、归一化方式（如LayerNorm/BatchNorm）、激活函数等。
- **数值范围**：输入数据的归一化参数（均值、标准差或最大最小值）。

#### 步骤2：维度对齐策略
根据差异类型，选择对应的适配方法：

| 差异类型 | 适配方法 | 实现建议与注意事项 |
| :--- | :--- | :--- |
| **变量集合不同** | 特征网络适配方法，如通过Conv | 通过卷积层改变变量数量维度，实现不同变量集合之间的数量对应。 |
| **空间分辨率/网格类型不同** | 空间插值 | **规则网格 ↔ 高斯网格**：使用双线性插值，注意经纬度边界（如经度周期边界）的处理。建议使用`torch.nn.functional.grid_sample`或`xarray`的插值方法。需验证守恒性（如质量、能量）是否被破坏。 |
| **时间结构不同** | 重塑 / 采样 / 聚合 | 使用`einops.rearrange`展平或堆叠时间步；若时间步数不一致，可选择最后一个时刻、平均或通过时间注意力聚合。 |
| **特征表示不同** | 特征投影 | 通过`nn.Conv2d` (1x1卷积) 或 `nn.Linear` 将特征通道数（即合并后的变量与垂直维）对齐。若涉及位置编码，需重算或对齐坐标。 |
| **张量维度顺序不同** | 维度重排 | 使用`einops.rearrange`或`torch.permute`统一为模块所需的布局。这是最基础且必需的步骤。 |

#### 步骤3：适配器设计模式
推荐使用**显式适配器（Explicit Adapter）**，将其设计为独立的`nn.Module`，而非修改原始模块内部代码。适配器应具备：
- **前向兼容性**：输入模块A的输出，输出模块B可接收的格式。
- **可微分性**：所有操作（插值、线性映射等）应支持梯度回传。
- **参数可选性**：某些映射（如变量线性组合）可设为可学习参数，以便在微调中优化。
- **透明性**：适配器内部应清晰记录每一步的变换，便于调试。

#### 步骤4：验证与测试
- **单元测试**：构造与模块输出形状一致的随机张量，单独测试适配器，断言输出形状与目标模块输入形状完全一致。
- **数值范围检查**：验证适配后的数据范围是否在目标模块的期望范围内（例如，目标模块可能要求输入经特定归一化）。
- **集成测试**：将组合后的模型在小型数据集（如单时间步、单样本）上运行，检查前向传播无维度错误，且损失函数能正常回传。
- **物理合理性检查**（可选）：对于关键变量（如温度、位势高度），验证插值前后的物理场分布是否合理。

## 4. 各模型典型维度特征与适配要点

### 4.1 Fuxi
- **典型张量布局**：`(B, T, V, H, W)`，其中 `T` 通常为2（过去+当前），`V≈70（包含陆地和大气变量），`H=721`，`W=1440`（高斯网格）。
- **特殊点**：使用高斯网格（纬度非均匀分布），变量包含“总降水量”等累积量，在自回归预测时需注意时间差分计算。
- **适配注意**：
    - **变量维度大**：适配时若目标模型变量少，需明确选择或通过可学习的权重矩阵进行降维。
    - **空间插值**：高斯网格到规则网格或均匀网格的插值需谨慎处理纬度权重，避免引入误差。

### 4.2 Pangu-Weather
- **典型张量布局**：`(B, T, V, H, W)`，`T=1`（当前时刻，默认压缩该维度），`V=5*13`（高空，如温度、位势）+ `5`（地面，如2米温度、10米风速），`H=721`，`W=1440`（高斯网格）。
- **特殊点**：高空与地面变量虽在同一张量中，但在内部处理时可能使用不同的嵌入方式。针对陆地变量使用2D位置编码，大气变量使用3D位置编码。
- **适配注意**：
    - **变量分离**：若组合模块需要高空/地面分离处理，需在适配器中切割或合并变量维。
    - **位置编码对齐**：当与其他模型拼接时，需确保坐标系统（如经度起点、纬度方向）一致，否则需重新计算或调整位置编码。

### 4.3 FourCastNet
- **典型张量布局**：`(B, T, V, H, W)`，`T` 通常为1（当前时刻，默认压缩该维度），`V=20`（整合了垂直层和原始变量），`H=720`，`W=1440`（规则等经纬网格）。
- **特殊点**：无显式垂直维度；使用规则网格，其AFNO（自适应傅里叶神经算子）模块强依赖于快速傅里叶变换（FFT），要求输入空间为等距网格。
- **适配注意**：
    - **引入垂直层**：若上游模块输出带垂直层，需在适配器中将垂直层展平或通过加权融合并入变量维。
    - **空间网格转换**：若上游模块输出为高斯网格，必须插值到规则网格，以匹配FourCastNet的FFT要求。插值时注意经度周期性边界。

### 4.4 GraphCast
- **典型张量布局**：`(B, V, L, H, W)`（或类似，时间维通过网格节点特征处理），使用均匀经纬网格 `(H=721, W=1440)`，垂直层 `L=37`（13层等压面+地面及更多层次）。
- **特殊点**：基于图神经网络（GNN），网格节点间的连接关系是固定的。其编码器和解码器处理多变量和多层次。
- **适配注意**：与基于CNN或Transformer的模型组合时，需通过空间插值来适配其固定网格，同时需确保变量和垂直层的对齐。

## 5. 实战工作流示例

**场景**：使用Fuxi编码器 + 适配器 + pangu解码器，构建一个混合大气预测模型。

1.  **模块接口分析**：
    - **Fuxi编码器期望输入**：`(B, T=2, V=70, H=721, W=1440)`，高斯网格。
    - **Fuxi编码器预计输出**：`([B, Dim=1536, t=T//P[0]=1, h=H//P[1]=180, w=W//P[2]=360])`，其中，P为Patch_size
    - **Pangu解码器期望输入**：`[B, c*h*w=C‘*H’*W‘=651600, Dim=192)`，其中，C'=C//P[0], H'=H//P[1], W'=W//P[2]，P为Patch_size
2.  **适配层：Fuxi编码器 → 维度适配器**
    - **目标**：将Fuxi编码器的输出维度与Pangu解码器期望输入维度进行匹配。
    - **操作**：
        - **变量映射**：Fuxi编码器输出为(B, Dim, t, h, w)，Pangu期望得到的输入为(B, c x h x w, Dim)，构建(t, h, w)->(c, h, w)的映射，以及构建2个Dim之间的映射。
        - **维度重排**：确认张量布局，将Dim转至最后一维，并且将Fuxi编码器输出后3维整合为1个维度。
        - **空间网格**：两者均使用相同的高斯网格 `(721, 1440)`，无需插值。
    - **适配器1输出**：`(B, T=2, V_p=10, L=13, H=721, W=1440)`。
3.  **Pangu解码器器**：接收适配层输出，通过解码器以后，输出Pangu的69变量，若想得到原Fuxi的变量，则调整图像恢复方法的参数。

## 6. 错误处理与鲁棒性
- **维度不匹配断言**：在每个适配器入口处使用`assert`语句或条件判断，明确检查输入张量的形状是否符合预期，并给出清晰的错误信息。
- **数值范围检查**：记录并跟踪每个模块输入的归一化参数（如`mean`, `std`）。适配器应包含可选的**反归一化/再归一化**模块，以防源模块输出与目标模块输入归一化方式不同。
- **网格坐标对齐**：在进行空间插值时，显式定义并传递源网格和目标网格的经纬度坐标。对于经度周期边界，使用`padding_mode='border'`或`'circular'`（如果`grid_sample`支持）来处理。
- **NaN/Inf处理**：在适配器输出后，添加检查点，使用`torch.isnan`或`torch.isinf`检测数值异常，并抛出警告或进行截断。

## 7. 最佳实践
- **优先使用现有工具**：
    - **维度重排**：推荐使用 `einops` 库，其语法清晰、不易出错。
    - **空间插值**：对于规则网格，使用 `torch.nn.functional.grid_sample`；对于更复杂的网格转换（如高斯到规则），可借助 `xarray` 的 `interp` 方法，但需注意将其集成到PyTorch计算图中可能带来的性能开销。
- **模块化设计**：将变量映射、空间插值、垂直变换、时间重塑分别封装为独立的 `Adapter` 子类，便于组合、复用和单元测试。
- **文档化映射关系**：在每个适配器类的docstring中，详细记录输入输出规格、映射逻辑（如变量对应表、插值方式）以及设计决策的理由。
- **支持动态配置**：适配器的关键行为（如变量名映射、插值目标分辨率、是否使用可学习投影）应通过构造函数参数传入，允许用户通过配置文件进行灵活配置，避免硬编码。
- **与原始模型解耦**：适配器不应依赖于原始模型的内部实现细节（如私有属性），仅依赖其公开的输入输出接口。

## 8. 输出格式
当用户提出具体的模块组合需求时，本Skill应指导生成以下内容：
1.  **模块接口分析表**：清晰列出每个待组合模块的名称、输入/输出张量形状（含维度顺序）、变量列表、网格类型等信息。
2.  **适配方案图**：用Mermaid流程图或类似方式展示数据流，标注每个适配器的位置、输入输出形状及其核心操作。
3.  **核心适配器代码骨架（PyTorch示例）**：提供至少一个关键适配器的代码实现，包含`__init__`和`forward`方法，并体现上述设计原则。
4.  **验证清单**：包括形状检查点、数值测试项（如插值前后的和守恒性）以及单元测试的建议。

# 模型复现技能

## 目标
基于论文的结构化知识总结和提供的组件库，进行深度学习模型的复现实现。

## 输入要求
1. **结构化知识** - 由前置任务提供的论文知识总结（包含模型架构、关键技术、超参数等）
2. **组件库** - 可用的模块和组件列表
3. **复现目标** - 需要复现的具体模型或方法

## 执行流程

### 1. 知识解析阶段
- 读取前置任务生成的结构化知识文件
- 提取关键信息：
  - 模型架构（层结构、连接方式）
  - 核心算法和技术点
  - 损失函数和优化器配置
  - 数据处理流程
  - 超参数设置

### 2. 组件匹配阶段
- 分析可用组件库的功能
- 将模型需求映射到现有组件：
  - 基础层（卷积、全连接、注意力等）
  - 激活函数和归一化层
  - 损失函数
  - 优化器
  - 数据加载器
- 识别需要自定义实现的部分

### 3. 架构设计阶段
- 设计模型类结构
- 规划模块组合方式
- 确定前向传播流程
- 设计训练和评估接口

### 4. 代码生成阶段
- **严格遵循** `code_standard_skill.md` 中的代码规范
- 优先使用提供的组件
- 对于无法用现有组件实现的部分，编写最小化的自定义代码
- 生成内容包括：
  - 模型定义文件
  - 训练脚本
  - 配置文件
  - 必要的工具函数

### 5. 验证阶段
- 检查代码完整性
- 验证组件使用的正确性
- 确认与论文描述的一致性
- 提供测试建议

## 输出规范

### 文件组织
```
model_name/
├── model.py          # 模型定义
├── train.py          # 训练脚本
├── config.py         # 配置文件
├── utils.py          # 工具函数（如需要）
└── README.md         # 使用说明
```

### 代码要求
1. **组件优先** - 最大化使用提供的组件库
2. **代码标准** - 遵循 `code_standard_skill.md` 规范
3. **最小实现** - 只编写必要的代码，避免冗余
4. **清晰注释** - 关键部分添加简洁注释
5. **可复现性** - 包含完整的超参数和随机种子设置

### 文档要求
- 模型架构说明
- 组件使用清单
- 训练命令示例
- 与原论文的对应关系

## 注意事项
1. 始终优先考虑使用现有组件，避免重复造轮子
2. 自定义代码应保持最小化和模块化
3. 确保生成的代码可以直接运行
4. 超参数应与论文保持一致
5. 如遇组件不足，明确指出缺失部分

## 依赖技能
- 前置：论文知识总结技能（提供结构化知识）
- 参考：`code_standard_skill.md`（代码规范）
- 参考：`component_depence_analyse_skill.md`（组件依赖分析技能）
- 其它根据有Agent自行分析需要提供的技能库中的哪些技能
